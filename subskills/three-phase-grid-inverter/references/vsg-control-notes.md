## VSG Control Architecture

VSG is a grid-forming control strategy that mimics synchronous generator behavior. Unlike PLL-based grid-following control, VSG generates its own angle and voltage reference.

### Control Path

```
Power Reference (Pref, Qref)
    -> Swing Equation (virtual inertia J, damping D)
    -> Angular frequency omega
    -> Angle theta (from integrating omega)
    -> Voltage Reference (from reactive power droop or direct Q control)
    -> dq/abc Transform
    -> PWM Modulation (SPWM or SVPWM)
    -> Gate Signals -> Inverter Bridge
```

### Swing Equation

The core of VSG control:

```
J * (dω/dt) = Pm - Pe - D * (ω - ω0)
```

Where:
- `J` = virtual inertia constant (kg·m² or equivalent)
- `D` = damping coefficient
- `Pm` = mechanical power reference (Pref)
- `Pe` = electrical power (measured)
- `ω` = angular frequency
- `ω0` = nominal angular frequency (2π×50 or 2π×60)

### Self-Synchronization

VSG inverters do not need PLL. The angle is generated by the swing equation, and power balance maintains synchronization with the grid.

- The angle theta is obtained by integrating omega: `theta = integral(omega)`
- Power balance is maintained through the virtual inertia and damping coefficients
- In weak grid or islanded conditions, VSG provides voltage and frequency support

### VSG vs PLL-based Control

| Aspect | VSG (Grid-forming) | PLL-based (Grid-following) |
|--------|-------------------|---------------------------|
| Synchronization | Self-synchronized via swing equation | Tracks grid via PLL |
| Angle source | Internal integration | PLL output |
| Weak grid performance | Better, provides voltage support | May lose synchronization |
| Inertia | Virtual inertia (J parameter) | No inherent inertia |
| Islanding capability | Can operate in island mode | Requires grid reference |

## PI Controller Tuning

### Anti-Windup Mechanisms

Back-calculation anti-windup is common in S-Function implementations:

```c
// Back-calculation anti-windup
*integrator = y_sat - kp * error;
```

**Critical Issue**: When `kp * error` is large, the integrator gets locked at the negative limit and cannot recover. This is because the back-calculation forces the integrator to compensate for the saturated output.

**Symptoms**:
- Output stuck at lower limit for extended periods
- System appears to never reach steady state
- Integrator value remains at negative saturation

**Solutions**:
1. Increase KI to speed up integrator recovery
2. Use conditional integration (freeze integrator when output is saturated)
3. Limit the back-calculation term: `*integrator = max(y_sat - kp*error, integrator_min)`
4. Use a separate anti-windup gain: `*integrator = y_sat - kaw * (y_unsat - y_sat)`

### Feedforward Priority

The main control quantity should use physical formula direct calculation (feedforward) as much as possible. PI is only for fine-tuning.

**Example**: Grid-connected current reference calculation

```
id_ref = Pref / (k * Vd)     % k = 1.5 (Clarke amplitude-invariant)
iq_ref = -Qref / (k * Vd)    % sign depends on Q convention
```

Where:
- `k` = 1.5 for Clarke amplitude-invariant transform (2/3 coefficient)
- `Vd` = d-axis voltage (measured)
- The negative sign in iq_ref depends on the Q sign convention used

**Key Rule**: If PI is struggling to correct a large steady-state error, the feedforward formula or parameters are wrong. Do not increase PI gains to compensate for feedforward errors.

### Integral Gain Recovery Time

When KI is too small, integrator recovery time may far exceed simulation duration.

**Example Calculation**:
- KI = 10, TS = 1e-4
- Integrator at -40, target is 0
- Recovery time = 40 / (KI * TS_step) = 40 / (10 * 1e-4) = 40,000 steps = 4 seconds

If simulation stop time is shorter than this, the system will appear to never reach steady state.

### PI Parameter Selection Guidelines

| Parameter | Typical Range | Effect of Too Small | Effect of Too Large |
|-----------|---------------|--------------------|--------------------|
| Kp | 0.1-10 | Slow response, large steady-state error | Oscillation, noise amplification |
| Ki | 1-100 | Slow integrator recovery, offset | Overshoot, instability |
| TS | 1e-6 to 1e-4 | Computation burden | Aliasing, instability |

## Power Calculation Verification

### Clarke Transform Coefficients

Two common conventions:

1. **Amplitude-invariant**: coefficient = 2/3
   - `iα = (2/3) * (ia - 0.5*ib - 0.5*ic)`
   - `iβ = (2/3) * (sqrt(3)/2 * ib - sqrt(3)/2 * ic)`

2. **Power-invariant**: coefficient = sqrt(2/3)
   - `iα = sqrt(2/3) * (ia - 0.5*ib - 0.5*ic)`
   - `iβ = sqrt(2/3) * (sqrt(3)/2 * ib - sqrt(3)/2 * ic)`

### P/Q Calculation Formulas

Using alpha-beta components:

```
P = k * (vα * iα + vβ * iβ)
Q = k * (vβ * iα - vα * iβ)
```

Where `k` depends on:
- Clarke coefficient (2/3 or sqrt(2/3))
- Voltage type (line-to-line or phase-to-ground)

### V-I Measurement Configuration

**Critical**: The V-I Measurement block output type affects all downstream calculations.

- **Phase-to-phase (line voltage)**: Vab, Vbc, Vca
- **Phase-to-ground (phase voltage)**: Va, Vb, Vc

If the model uses line voltage but the control algorithm expects phase voltage, a conversion is needed:

```
va = (vab - vca) / 3
vb = (vbc - vab) / 3
vc = (vca - vbc) / 3
```

### Verification Checklist

When S-Function internal P/Q disagrees with Scope, check in order:

1. **V-I Measurement voltage mode**: phase-to-phase or phase-to-ground?
2. **Clarke transform coefficient**: amplitude-invariant (2/3) or power-invariant (sqrt(2/3))?
3. **P/Q formula**: correct alpha-beta components and sign convention?
4. **Sign convention**: Q sign depends on iα/iβ ordering
5. **Line-to-phase conversion**: if needed, verify conversion formulas

### Common Mismatch Symptoms

| Symptom | Likely Cause |
|---------|--------------|
| P deviation ~1.5x | Line/phase voltage confusion, or wrong Clarke coefficient |
| Q sign reversed | Alpha/beta order or sign error in Q formula |
| P/Q both very small | Input port not connected (floating) |
| P/Q oscillating | Angle or measurement error |

## S-Function Implementation Notes

### Time Scale Separation

The S-Function internal TS must match the control algorithm step, not the simulation step.

- Simulation step: typically 1e-6 to 1e-5 (for power electronics switching)
- Control algorithm step: typically 1e-4 to 1e-3 (for PI controllers)
- S-Function TS: must match control algorithm step

### Data Reading from Scope

When reading simulation results:

```matlab
% Clear workspace first
evalin('base', 'clear');

% Run simulation
simOut = sim('model_name', 'StopTime', 'N');

% Read Scope data
scope_data = simOut.scope_variable_name;
t = scope_data.time;
P = scope_data.signals(1).values;
Q = scope_data.signals(2).values;

% Steady-state analysis
idx = t >= steady_start & t <= steady_end;
fprintf('P mean: %.2f\n', mean(P(idx)));
fprintf('Q mean: %.2f\n', mean(Q(idx)));
```

**Important**: `fieldnames(simOut)` may not show all logged signals. Use direct property access.

### Common S-Function Issues

1. **MEX compilation not taking effect**: Old MEX binary is locked or cached
   - Fix: `clear <sfunc_name>` then re-mex

2. **Input port not connected**: Signal always zero
   - Fix: Check S-Function input wiring in model

3. **Wrong angle reference**: dq transform produces incorrect results
   - Fix: Verify angle source (PLL output vs VSG theta)

4. **Coefficient mismatch**: Internal calculations disagree with Scope
   - Fix: Verify Clarke/Park coefficients match between S-Function and model blocks
